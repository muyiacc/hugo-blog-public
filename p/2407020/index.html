官网：https://rclone.org/

安装

To install rclone on Linux/macOS/BSD systems, run:

1

sudo -v ; curl https://rclone.org/install.sh | sudo bash

For beta installation, run:

1

sudo -v ; curl https://rclone.org/install.sh | sudo bash -s beta

新增配置

要实现后续操作，第一次使用rclone首先需要增加远程存储配置

新增一个远程配置，这是一个交互式的命令，执行后按照提示操作即可

1

rclone config

挂载

将远程存储挂载到本地文件夹

比如，将坚果云挂载 /home/taoone/jianguoyun

1

mkdir -p /home/taoone/mnt/jianguoyun

1

rclone mount --buffer-size 1M --vfs-cache-mode full jianguoyun: /home/taoone/jianyuoyun --daemon

取消挂载

1

fusermount -u /home/taoone/jianyuoyun

挂载alist到本地

1

mkdir -p ~/alist

前台挂载

1

rclone mount --buffer-size 4M --vfs-cache-mode writes alist: ~/alist

后台挂载

1

rclone mount --buffer-size 4M --vfs-cache-mode writes alist: ~/alist --daemon

取消挂载

1

fusermount -u ~/alist

写入定时任务，实现开机挂载

1

crontab -e

1
2

# 开机挂载 rclone alist
@reboot rclone mount --buffer-size 4M --vfs-cache-mode writes alist: ~/alist --daemon

同步（谨慎使用）

单向同步（谨慎使用）

注意：这可能导致数据丢失，首先使用 --dry-run or the --interactive/-i测试

官方文档：https://rclone.org/commands/rclone_sync/

1

rclone sync --interactive SOURCE remote:DESTINATION

案例1：从远程坚果云同步到本地

1

rclone sync jianguoyun:/ /home/taoone/jianyunyun

案例2：从本地同步到远程坚果云（本地需要存在）

1

rclone sync /home/taoone/jianyunyun jianguoyun:/

双向同步（谨慎使用）

！！！暂处于测试阶段，谨慎使用！！！

官方文档：https://rclone.org/commands/rclone_bisync/

1

rclone bisync remote1:path1 remote2:path2 [flags]

复制

将文件从源复制到目标位置，跳过相同的文件

1

rclone copy source:sourcepath dest:destpath

官方文档：https://rclone.org/commands/rclone_copy/

案例1：将本地software文件夹复制到alist下aliyunpan/software

1
2

# 复制（复制到远程可能需要等待完成）-P可查看具体信息
rclone copy ~/software alist:/aliyunpan/software -P

复制可当作备份使用，写为脚本文件，需要时就运行一下。

1

vim ~/bin/rclone_copy_local_software_to_alist_aliyunpan_software.sh

脚本的文件含义为：rclone从本地复制software文件夹到alist下的aliyunpan/software

1
2

#!/bin/bash
rclone copy ~/software alist:/aliyunpan/software

1

chmod +x ~/bin/rclone_copy_local_software_to_alist_aliyunpan_software.sh

提示：

alist有些盘可以妙传，但是超过一定大小将不可用，为了秒传的可用性，在备份时可以添加 --max-size参数跳过最大的文件，比如 --max-size 500M跳过超过500M的文件

案例：

1

rclone copy alist:aliyunpan/infotech/ alist:115/infotech/ --max-size 500M -P

备份

挂载和复制可以同时使用，就比如我挂载了alist到本地，我同样可以使用复制功能使用该挂载的文件夹

1
2

# 挂载
rclone mount --buffer-size 1M --vfs-cache-mode full alist: ~/alist --daemon

系统服务

note 暂时有问题

配置系统服务，实现开机自动挂载

命令预览：

1
2
3

rclone mount \
    --vfs-cache-mode full \
    alist: /mnt/alist

参数解释： 1. - minimal: 最小缓存，只缓存打开的文件。 - writes: 缓存写入，适合频繁写入的场景。 - full: 完全缓存，适合频繁读取的场景。

创建一个新的systemd服务文件：

1

sudo vim /etc/systemd/system/rclone-mount.service

文件中添加以下内容：

[Unit]
Description=Rclone Mount Service
Wants=network-online.target
After=network-online.target

[Service]
Type=simple
ExecStart=/usr/bin/rclone mount \
    alist:/ /mnt/alist \
    --dir-cache-time 2h \
    --dir-cache-size 1G \
    --poll-interval 10m \
    --use-mmap \
    --vfs-cache-mode full \
    --vfs-read-chunk-size 100M \
    --vfs-read-chunk-size-limit 128M \
    --vfs-cache-max-age 2h \
    --vfs-cache-poll-interval 10m \
    --vfs-cache-chunk-size 100M
ExecStop=/usr/bin/fusermount -u /mnt/alist

[Install]
WantedBy=multi-user.target

创建挂载点

1

sudo mkdir -p /mnt/alist

启动服务

1
2

sudo systemctl daemon-reload
sudo systemctl start rclone-mount.service

检查服务的状态，确保其正常运行

1

sudo systemctl status rclone-mount.service

设置服务开机自启

1

sudo systemctl enable rclone-mount.service

重启服务

1

sudo systemctl restart rclone-mount.service
